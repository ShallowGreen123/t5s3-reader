## 1 项目概述

### 1.1 项目背景

CrossPoint Reader 是一款基于 ESP32-C3 微控制器的开源电子书阅读器项目，专为电子墨水屏（E-Ink）设备设计。项目采用模块化架构，集成了文件管理、网络通信、云端同步等核心功能，支持多种电子书格式，旨在为用户提供简洁、高效的阅读体验。

本项目区别于传统电子书阅读器的核心特点在于：其一，采用开源固件架构，允许用户深度定制和二次开发；其二，集成 KOReader 云端同步服务，实现跨设备阅读进度同步；其三，支持 WebDAV 协议，可与主流图书管理软件（如 Calibre）无缝对接；其四，内置 HTTP/WebSocket 服务器，提供便捷的文件传输和管理接口。

### 1.2 核心功能定位

本项目的核心功能可归纳为以下四个层面：

**阅读功能层面**：支持 EPUB、TXT、XTC 等主流电子书格式的解析与渲染，提供流畅的翻页体验和丰富的阅读辅助功能（如字体大小调节、连字处理、断词换行等）。

**文件管理层面**：支持 SD 卡存储管理，提供本地文件系统浏览、文件上传下载、目录创建删除等基础操作能力。

**网络功能层面**：支持 WiFi 站点模式（STA）和热点模式（AP）两种网络接入方式，内置 HTTP 服务器和 WebSocket 服务，支持 WebDAV 协议，可接入 OPDS 图书服务器，实现云端图书同步。

**系统管理层面**：提供完善的系统设置功能，包括界面主题定制、按钮映射配置、睡眠模式管理、语言切换、固件 OTA 更新等。

### 1.3 技术架构概览

本项目采用分层架构设计，各层职责明确，模块间通过定义清晰的接口进行交互。硬件抽象层（HAL）封装了底层硬件操作细节，业务逻辑层专注于电子书解析、页面渲染、状态管理等核心功能，应用层通过 Activity 机制实现用户交互流程的编排。网络通信层独立实现各类网络协议，确保功能模块的内聚性和可维护性。

## 2 核心功能模块分析

### 2.1 硬件抽象层模块

硬件抽象层（Hardware Abstraction Layer，HAL）是整个系统的基础层，负责封装 ESP32-C3 微控制器的底层硬件操作，为上层提供统一的硬件访问接口。该模块包含以下核心子模块：

**显示驱动模块（HalDisplay）**  
显示驱动模块负责管理电子墨水屏（SSD1677）的显示输出。电子墨水屏相较于传统液晶屏具有功耗低、阳光下可读性强、断电后保持显示内容等特点，但其刷新速度较慢，需要特殊的驱动策略。该模块实现了以下核心功能：帧缓冲管理（支持单缓冲模式）、全屏刷新控制、局部刷新优化、显示方向调节（支持纵向、横向、倒置四种模式）。在性能方面，该模块通过 `EINK_DISPLAY_SINGLE_BUFFER_MODE` 编译参数优化内存使用，确保在资源受限的嵌入式环境中稳定运行。

**通用输入输出模块（HalGPIO）**  
通用输入输出模块负责管理设备上的物理按钮输入。设备配置了正面按钮组（返回、确认、左方向、右方向）和侧面按钮组（上一页、下一页），以及电源按钮。该模块实现了按钮状态检测、按钮按下时长统计、多按钮组合识别等功能。通过与 `MappedInputManager` 的配合，支持按钮功能重映射，满足不同用户的使用习惯偏好。模块还实现了电源按钮长按检测逻辑，用于区分短按睡眠和长按关机等不同操作。

**电源管理模块（HalPowerManager）**  
电源管理模块负责设备供电系统的监控与管理。该模块实现了电池电量监测功能，可实时获取电池电压并估算剩余电量，为用户提供电量提示。同时，模块负责管理设备的睡眠和唤醒流程，支持多种睡眠触发条件（超时自动睡眠、按钮触发睡眠等），并在唤醒时进行电源按钮按压时长的校验，确保唤醒操作的可靠性。

**存储管理模块（HalStorage）**  
存储管理模块封装了 SD 卡存储的操作接口。设备支持 SD 卡热插拔，模块负责 SD 卡的初始化挂载、文件系统操作、存储空间查询等。在文件系统层面，模块支持长文件名（UTF-8 编码），确保多语言书名的正确显示。模块还实现了电子书缓存目录的管理，包括元数据缓存、封面图片缓存、分页数据缓存等，确保二次打开书籍时的快速加载。

**系统管理模块（HalSystem）**  
系统管理模块提供了 ESP32 微控制器的基础服务接口，包括看门狗定时器管理、系统重启控制、唯一设备标识获取等功能。该模块是其他 HAL 模块的基础，负责系统时钟初始化和外设资源配置。

### 2.2 电子书解析与渲染模块

电子书解析与渲染模块是本项目的核心功能模块，负责将电子书文件转换为可供显示的页面内容。该模块支持多种电子书格式，具有高度的可扩展性。

**EPUB 解析模块（lib/Epub）**  
EPUB 是当前最主流的电子书格式标准，本项目对 EPUB 格式提供了完善的解析支持。EPUB 文件本质上是 ZIP 压缩包，内部包含 XML 格式的元数据、内容文档（HTML/XHTML）、样式表（CSS）、图片资源等。解析流程包含以下关键环节：

元数据解析阶段负责读取 OPF 文件（content.opf），提取书名、作者、语言、出版商等基础信息，同时解析 OPF 文件中定义的 spine（阅读顺序）和 manifest（资源清单）。目录解析阶段支持 NCX 格式（EPUB 2）和 NAV 格式（EPUB 3）两种目录文件格式，将章节标题与内容资源建立映射关系。内容解析阶段负责解析 HTML/XHTML 文档，提取文本内容和内联图片引用，处理 CSS 样式信息。缓存构建阶段将解析结果以二进制格式（book.bin）存储到缓存目录，包含元数据、spine 列表、toc 列表等结构化数据，加速二次加载。

EPUB 模块还包含高级排版功能：连字处理（Hyphenation）支持多语言自动断词，包括德语、英语、西班牙语、法语、意大利语、俄语、乌克兰语等语言包；CSS 样式解析支持文本对齐、缩进、字间距、行高等常见样式属性；图片渲染支持 PNG、JPEG 等格式的解码和显示。

**TXT 解析模块（lib/Txt）**  
TXT 纯文本格式虽然结构简单，但因其通用性强仍被广泛使用。TXT 解析模块负责将纯文本文件按固定宽度进行分页处理，支持多种编码格式（UTF-8、GBK 等）的自动识别。模块在解析时会对文本进行简单的段落识别和排版处理，提升阅读体验。

**XTC 解析模块（lib/Xtc）**  
XTC 是项目自定义的电子书格式，采用二进制编码，包含预先生成的页面数据（section.bin）和元数据（book.bin）。这种格式的优势在于解析速度快、无需运行时计算，适合对加载性能有较高要求的场景。XTC 文件的生成工具不在本项目范围内，但解析器的实现确保了与标准工具生成文件的兼容性。

**页面渲染引擎（lib/GfxRenderer）**  
页面渲染引擎负责将解析后的文本和图片数据绘制到电子墨水屏的帧缓冲区。渲染引擎的核心设计考量包括：

字体管理方面，引擎集成了 EpdFont 字体渲染库，支持多种内置字体（NotoSerif、NotoSans、OpenDyslexic），并实现了字体缓存管理（FontCacheManager）以优化内存使用。图像渲染方面，引擎集成了 PNG 和 JPEG 格式的解码器，支持图片到帧缓冲的转换，并实现了抖动算法（Dither）以适配电子墨水屏的灰度显示能力。排版计算方面，引擎实现了文本分行算法，考虑单词断词、标点避头尾、孤儿寡行控制等排版规则，生成符合阅读习惯的页面布局。

渲染引擎还支持分页缓存预计算，在用户阅读当前页时预先渲染后续页面，缩短翻页响应时间。刷新策略方面，引擎支持全屏刷新和局部刷新两种模式，可配置每 N 页进行一次全屏刷新，以平衡显示质量和刷新速度。

### 2.3 网络通信模块

网络通信模块为本设备提供了与外部系统交互的能力，是实现文件传输、云端同步等高级功能的基础。

**WiFi 管理模块**  
WiFi 管理模块实现了 IEEE 802.11 无线网络的接入功能，支持站点模式（STA）和热点模式（AP）两种工作方式。

在站点模式下，设备作为无线终端接入现有 WiFi 网络，需要提供有效的 SSID 和密码。模块实现了 WiFi 网络扫描、信号强度检测、自动重连等功能，支持保存多个网络配置（WifiCredentialStore）。设备支持 WEP、WPA、WPA2 等常见认证方式。

在热点模式下，设备作为无线接入点广播自身网络，允许其他设备直接连接。热点模式适用于无现有 WiFi 网络或网络环境受限时使用，热点默认配置为开放网络（无密码），SSID 默认为 "CrossPoint-Reader"。

**HTTP 服务器模块（lib/network/CrossPointWebServer）**  
HTTP 服务器是本项目网络功能的核心组件，运行在 TCP 端口 80，提供文件管理和设备状态查询的 RESTful 接口。

服务器端点定义如下：

| 端点 | 方法 | 功能描述 |
|------|------|----------|
| `/` | GET | 返回文件管理 Web 界面首页 |
| `/files` | GET | 返回文件浏览器 Web 界面 |
| `/api/status` | GET | 返回设备状态信息（版本、IP、内存、上传时间等） |
| `/api/files` | GET | 返回指定目录下的文件列表，支持 path 参数 |
| `/upload` | POST | 上传文件到 SD 卡，支持 multipart/form-data 格式 |
| `/mkdir` | POST | 在指定目录下创建新文件夹 |
| `/delete` | POST | 删除指定的文件或文件夹 |

文件列表接口返回 JSON 格式数据，包含文件名称、大小、是否为目录、是否为 EPUB 格式等属性。文件上传接口支持断点续传场景下的文件覆盖，支持指定上传目标目录。服务器自动过滤隐藏文件（以"."开头）和系统文件夹（"System Volume Information"、"XTCache"）。

**WebSocket 服务器模块**  
WebSocket 服务器运行在 TCP 端口 81，提供高速二进制数据传输通道。与 HTTP 的请求-响应模式不同，WebSocket 建立了客户端与服务器的持久连接，支持服务器主动推送数据。在文件传输场景下，WebSocket 可用于实现更高效的增量上传和实时进度反馈。

**WebDAV 处理器（lib/network/WebDAVHandler）**  
WebDAV（Web-based Distributed Authoring and Versioning）是 HTTP 的扩展协议，定义了如何在 Web 上进行分布式创作和版本管理。本项目实现了 WebDAV 协议的子集，以支持与 Calibre 等图书管理软件的集成。

支持的 WebDAV 方法包括：OPTIONS（获取服务器能力）、PROPFIND（查询资源属性）、GET（下载文件）、HEAD（获取资源头信息）、PUT（上传文件）、DELETE（删除资源）、MKCOL（创建集合/目录）、MOVE（移动资源）、COPY（复制资源）、LOCK/UNLOCK（锁定/解锁资源）。WebDAV 处理器在收到文件变更请求时，会自动清除相关电子书的缓存，确保下次打开时加载最新内容。

**Calibre 连接模块（src/activities/network/CalibreConnectActivity）**  
Calibre 是最流行的开源电子书管理软件，支持通过 HTTP 方式访问其内置的图书库。本模块提供了与 Calibre 的集成入口，启动后在设备端显示连接指南，指导用户在 Calibre 端配置内容服务器并连接。连接建立后，用户可通过设备浏览和下载 Calibre 图书库中的书籍。

**OPDS 解析器（lib/OpdsParser）**  
OPDS（Open Publication Distribution System）是一种基于 Atom 的电子书目录格式，被众多出版商和网络书库采用。本项目实现了 OPDS 目录的解析器（OpdsParser）和流式读取器（OpdsStream），支持设备浏览 OPDS 书库、查看书籍元数据、下载书籍等操作。

**KOReader 同步模块（lib/KOReaderSync）**  
KOReader 是另一款流行的开源电子书阅读器，其云端同步服务（sync.koreader.rocks）允许用户跨设备同步阅读进度。本项目实现了与 KOReader 同步服务的对接，包括认证（基于用户名和密码 MD5 哈希的认证头）、进度查询（GET /syncs/progress/:document）、进度更新（PUT /syncs/progress）等接口。

同步流程为：首先计算电子书的文档哈希（基于文件内容的 MD5），然后查询云端是否存在该文档的进度记录，若存在则返回进度数据（XPath 格式的位置信息），用户可选择应用云端进度或上传本地进度。进度数据包含当前章节索引、页码、段落索引等精确定位信息。

**HTTP 下载器（lib/network/HttpDownloader）**  
HTTP 下载器提供了通用的 HTTP/HTTPS 下载功能，支持文件下载、断点续传、进度回调等。该模块被 OPDS 浏览、OTA 固件更新等功能复用。

### 2.4 用户界面模块

用户界面模块负责构建与用户的交互流程，采用 Activity（活动）机制组织界面和交互逻辑。

**Activity 框架（src/activities）**  
Activity 框架是本项目 UI 架构的核心，借鉴了 Android 的 Activity 概念。每个 Activity 代表一个独立的用户界面和交互流程，如阅读界面、设置界面、文件浏览界面等。Activity 之间可以相互启动和传递结果，形成导航栈（Activity Stack）。

框架核心组件包括：Activity 基类定义了 onEnter（进入时调用）、onExit（退出时调用）、loop（主循环，用于处理输入和状态更新）、render（渲染界面）等生命周期方法；ActivityManager 负责管理 Activity 栈、处理输入事件分派、协调渲染时序；RenderLock 机制确保渲染操作的原子性，避免显示撕裂。

**主屏幕活动（HomeActivity）**  
HomeActivity 是设备的默认首页，显示最近阅读书籍列表、系统快捷入口（设置、WiFi、文件浏览）、睡眠/关机按钮等。首页采用卡片式布局，展示书籍封面、标题、作者、阅读进度等信息。用户可通过按钮导航选择进入阅读界面或执行其他操作。

**文件浏览活动（FileBrowserActivity）**  
FileBrowserActivity 提供 SD 卡文件的图形化浏览界面。界面顶部显示当前路径，可通过导航按钮进入上级目录或子目录。文件列表支持按名称、大小、日期排序，支持 EPUB、TXT、XTC 等格式的筛选。选中文件后可执行打开、删除、重命名等操作。

**阅读器活动（ReaderActivity 系列）**  
阅读器活动是用户进行阅读的核心界面，包含多个派生类以适配不同电子书格式：

EpubReaderActivity 是最完善的阅读器实现，提供完整的 EPUB 阅读功能，包括页面渲染、翻页、章节跳转、进度显示、书签、字体设置、目录访问、脚注浏览等。TxtReaderActivity 和 XtcReaderActivity 分别针对 TXT 和 XTC 格式进行了适配。

阅读器活动支持多种翻页操作方式：正面按钮翻页（确认/返回键）、侧面按钮翻页、电源按钮短按翻页（可配置）。阅读器界面底部可配置显示进度条（书籍进度、章节进度）、当前页码/总页数、章节标题等状态栏信息。

**设置活动（SettingsActivity 系列）**  
设置活动提供系统参数的配置入口，包括：

SleepActivity 提供睡眠/唤醒功能入口和睡眠壁纸选择。SettingsActivity 是主设置菜单，包含显示、阅读、网络、系统等子菜单入口。StatusBarSettingsActivity 允许用户自定义状态栏的显示内容、位置、样式。KOReaderSyncActivity 管理 KOReader 云端同步的认证和同步操作。KOReaderAuthActivity 处理 KOReader 账户的登录认证。KOReaderSettingsActivity 配置 KOReader 同步选项。OpdsServerListActivity 管理 OPDS 书库服务器列表。OpdsSettingsActivity 配置 OPDS 连接选项。LanguageSelectActivity 提供界面语言选择，支持 20 余种语言。ButtonRemapActivity 允许用户自定义按钮的功能映射。ClearCacheActivity 提供缓存清理功能，释放存储空间。OtaUpdateActivity 处理固件 OTA 更新检查和安装。

**网络活动（NetworkActivity 系列）**  
网络活动处理 WiFi 连接和文件传输相关的交互流程：

NetworkModeSelectionActivity 提供网络模式选择（加入现有网络、连接 Calibre、创建热点）。WifiSelectionActivity 扫描并显示可用 WiFi 网络，支持输入密码连接。CrossPointWebServerActivity 启动 HTTP 服务器并显示连接信息。CalibreConnectActivity 是 Calibre 连接模式的活动入口。OpdsBookBrowserActivity 提供 OPDS 书库的图形化浏览界面。

**工具活动（UtilActivity 系列）**  
工具活动提供辅助性交互组件：

ConfirmationActivity 显示确认对话框，用于危险操作的二次确认。FullScreenMessageActivity 显示全屏消息提示。KeyboardEntryActivity 提供文本输入键盘界面。BmpViewerActivity 提供位图图片的全屏查看功能。QrDisplayActivity 显示二维码（用于分享 WiFi 信息等）。

### 2.5 配置与状态管理模块

**设置管理（src/CrossPointSettings）**  
CrossPointSettings 是全局配置信息的单例管理器，采用静态实例模式，提供整个应用生命周期内持久化的配置参数存储。配置数据以 JSON 格式存储到 SD 卡的配置文件中，开机时自动加载。

配置项涵盖以下类别：

显示配置包括睡眠屏幕模式（暗色、亮色、自定义、封面、空白）、睡眠屏幕封面模式（适应、裁剪）、封面滤镜（无、黑白、反转黑白）、屏幕方向（纵向、横向CW、倒置、横向CCW）、状态栏模式（无、仅图标、全功能、仅书籍进度条、仅章节进度条）、状态栏进度条类型（书籍进度、章节进度、隐藏）、状态栏标题显示（书名、章节名、隐藏）、状态栏进度条粗细（细、正常、粗）、UI 主题（经典、Lyra、Lyra 3封面、圆角）。

阅读配置包括字体族系（NotoSerif、NotoSans、OpenDyslexic）、字体大小（小、中、大、超大）、行间距（紧凑、正常、宽松）、段落对齐（两端对齐、左对齐、居中、右对齐、跟随书籍）、EPUB 图片渲染模式（显示、占位符、隐藏）。

系统配置包括自动睡眠超时（1分钟、5分钟、10分钟、15分钟、30分钟）、电子墨水屏刷新频率（每1页、每5页、每10页、每15页、每30页）、电源按钮短按动作（忽略、睡眠、翻页、强制刷新）、电池图标显示策略（从不显示、阅读时隐藏、始终显示）。

按钮配置包括正面按钮布局（四种预设布局）、侧面按钮布局（上一页-下一页、下一页-上一页）、按钮功能重映射。

网络配置包括默认 WiFi 网络、最近使用的 WiFi 网络列表。

**状态管理（src/CrossPointState）**  
CrossPointState 管理应用运行时状态，包括最近打开的书籍路径、阅读器活动的加载计数、睡眠壁纸历史记录等。与持久化的配置不同，状态数据反映应用的运行时上下文。

**服务器配置存储（OpdsServerStore、WifiCredentialStore、KOReaderCredentialStore）**  
各网络服务的认证信息采用独立的加密存储机制。OpdsServerStore 管理 OPDS 书库的服务器地址和认证信息。WifiCredentialStore 管理 WiFi 网络的 SSID 和密码。KOReaderCredentialStore 管理 KOReader 同步账户的认证信息。认证信息在存储时经过混淆处理，增加安全性。

### 2.6 国际化模块

**国际化框架（lib/I18n）**  
国际化模块支持多语言界面，框架核心设计包括：语言资源文件（YAML 格式）存储各语言的翻译字符串；翻译键（Translation Key）标识每个字符串，而非硬编码的文本；动态语言切换无需重启应用。

当前支持的界面语言包括：英语、德语、法语、西班牙语、意大利语、葡萄牙语、荷兰语、波兰语、俄语、乌克兰语、捷克语、丹麦语、芬兰语、瑞典语、匈牙利语、罗马尼亚语、斯洛文尼亚语、土耳其语、白俄罗斯语、加泰罗尼亚语、哈萨克语、立陶宛语等。

## 3 用户端功能清单

### 3.1 阅读功能

用户端阅读功能是本产品的核心价值所在，涵盖从打开书籍到完成阅读的完整流程。

**书籍打开与加载**  
用户可通过以下方式打开书籍：从首页最近阅读列表选择、最近阅读活动选择、文件浏览器中选择、OPDS 书库中下载后选择。系统根据文件扩展名自动选择对应的阅读器活动。加载过程中显示加载进度，加载完成后自动定位到上次阅读位置。

**页面浏览**  
用户可通过多种方式翻页：使用正面按钮（确认键前进、返回键后退）、使用侧面按钮（上一页/下一页键）、短按电源按钮（需在设置中启用）、点击屏幕特定区域（如已启用触摸功能）。翻页响应时间应控制在 200 毫秒以内（含页面渲染和屏幕刷新）。

**进度管理**  
系统自动保存阅读进度，包括当前 spine 索引、spine 内页码、段落索引等信息。进度数据存储在书籍元数据缓存中。进度可通过状态栏直观显示，支持进度条形式或百分比形式。

**章节导航**  
用户可通过以下方式在章节间导航：从目录界面选择目标章节、跳转至指定百分比位置、向前/向后跳转固定章节数。目录界面显示章节标题和当前阅读位置标记。

**阅读辅助功能**  
字体设置支持选择字体族系（衬线、无衬线、 dyslexia 友好字体）、调节字体大小（4档）、调节行间距（3档）、调节段落对齐方式（5种）。脚注处理支持脚注链接识别和脚注内容浏览。图片浏览支持点击图片放大查看、页面内图片显示开关。连字处理支持多语言自动断词，可选择开启或关闭。

### 3.2 文件管理功能

**文件浏览**  
用户可在设备上浏览 SD 卡的文件系统。界面以列表形式显示当前目录内容，每项显示文件图标（区分文件夹、书籍文件、其他文件）、文件名称、文件大小或子项数量。用户可通过导航按钮进入子目录或返回上级目录。

**文件操作**  
用户可执行以下文件操作：打开文件（进入对应阅读器）、删除文件（需二次确认）、创建文件夹、重命名文件。文件操作支持批量选择模式。

**文件传输**  
用户可通过 HTTP 接口上传书籍文件到设备：连接到设备 IP 地址的 Web 界面，选择上传功能，选择本地 EPUB/TXT 文件，上传至目标目录。上传完成后可在设备上立即打开阅读。

**WebDAV 访问**  
支持通过 WebDAV 协议访问设备存储空间。用户可在 Calibre 等支持 WebDAV 的应用中配置设备地址，实现书籍的同步管理。

### 3.3 网络功能

**WiFi 连接**  
用户可将设备连接到 WiFi 网络：从网络列表中选择目标网络，输入密码（虚拟键盘输入），保存密码以备后续自动连接。连接状态实时显示，包括信号强度和 IP 地址。

**热点模式**  
用户可让设备创建 WiFi 热点供其他设备连接：进入热点模式，设备广播指定 SSID，其他设备连接后可通过 HTTP 接口传输文件。

**云端同步**  
用户可将阅读进度同步到 KOReader 云端服务：配置 KOReader 账户信息（用户名、密码），在阅读界面选择同步功能，系统显示本地和云端进度对比，用户选择应用云端进度或上传本地进度。

**OPDS 浏览**  
用户可浏览和下载 OPDS 格式的网络书库：添加 OPDS 服务器地址，浏览书库目录结构，查看书籍元数据和封面，选择并下载书籍到本地。

### 3.4 系统设置功能

**显示设置**  
用户可配置睡眠屏幕外观（壁纸、滤镜）、屏幕方向（支持旋转 90 度）、状态栏内容和样式、UI 主题。

**阅读设置**  
用户可配置默认字体、字体大小、行间距、对齐方式、图片显示策略。

**系统设置**  
用户可配置自动睡眠时间、刷新频率、电源按钮动作、按钮布局、语言设置。

**存储管理**  
用户可查看存储空间使用情况，清理缓存数据，检查固件更新。

## 5 关键业务流程

### 5.1 书籍阅读流程

书籍阅读是用户最核心的使用场景，涉及电子书解析、页面渲染、进度管理等多个环节。

**流程描述**  
用户从首页选择书籍或通过文件浏览器打开书籍文件。系统根据文件扩展名判断格式类型，实例化对应的阅读器 Activity。如果书籍首次打开或缓存已过期，系统执行 EPUB 解析流程：打开 ZIP 文件、解析 OPF 元数据、解析目录结构、构建 spine 映射、生成封面缩略图。如果缓存有效，系统直接从缓存加载元数据和分页数据。加载完成后，系统恢复上次阅读位置，开始渲染当前页面。用户翻页时，系统根据翻页方向计算目标页码，渲染目标页面内容，更新显示。如果翻页导致阅读位置变化，系统异步保存进度到缓存。系统根据配置的自动睡眠时间检测用户无操作时间，超时后进入睡眠状态。

**异常处理**  
解析失败时显示错误提示，提供返回和重试选项。SD 卡读取错误时尝试重新挂载，失败后提示检查存储设备。内存不足时尝试释放页面缓存，仍不足时提示关闭其他应用或重启设备。

### 5.2 文件上传流程

文件上传是用户向设备添加书籍的主要方式，涉及 HTTP 传输、文件存储、缓存管理等多个环节。

**流程描述**  
用户通过 Web 浏览器或 WebDAV 客户端连接到设备 HTTP 接口。用户选择要上传的文件，指定目标目录（可选）。客户端以 multipart/form-data 格式发送文件数据。服务器端接收数据流，写入 SD 卡的临时文件。传输完成后，服务器验证文件完整性，移到目标目录。如果文件是 EPUB 格式，系统自动检测并清除相关缓存。下次打开该书籍时，系统执行完整的解析和缓存构建流程。

**异常处理**  
存储空间不足时中断传输并返回错误。网络中断导致传输失败时，服务器清理临时文件。文件格式不支持时返回错误提示。文件名包含特殊字符时进行 URL 编码处理。

### 5.3 KOReader 同步流程

KOReader 同步使用户能够在多个设备间保持阅读进度一致，涉及文档哈希计算、云端 API 调用、进度映射转换等环节。

**流程描述**  
用户在阅读界面选择同步功能。系统计算当前电子书的文档哈希（基于文件内容的 MD5）。系统检查是否已配置 KOReader 账户，无账户时引导用户配置。系统调用 KOReader 同步 API 获取云端进度。系统比较本地进度和云端进度，显示差异供用户选择。用户选择应用云端进度或上传本地进度。系统执行相应的进度更新操作。如果用户选择应用云端进度，系统将云端的 XPath 格式位置转换为本地 spine 索引和页码。系统跳转到目标位置，继续阅读。

**异常处理**  
网络连接失败时提示检查网络设置。认证失败时提示检查账户信息。文档在云端不存在时提示无云端进度，可选择上传本地进度。云端进度格式无法解析时提示同步失败。

### 5.4 OPDS 浏览流程

OPDS 浏览使用户能够从网络书库发现和获取书籍，涉及目录解析、资源获取、本地存储等环节。

**流程描述**  
用户从设置中添加 OPDS 服务器地址。系统验证服务器可达性，获取根目录（catalog）内容。系统解析 OPDS feed，提取条目列表（书籍或子目录）。用户浏览目录结构，进入感兴趣的分类或搜索结果。系统解析条目详情，显示书籍元数据（标题、作者、封面、简介）。用户选择下载书籍。系统获取下载链接，下载文件到本地 SD 卡。下载完成后，系统清除缓存并返回文件浏览器。

**异常处理**  
服务器不可达时提示检查网络和地址配置。OPDS 格式解析失败时提示格式不支持。下载链接失效时尝试刷新目录或提示手动获取链接。存储空间不足时提示清理存储。

### 5.5 OTA 固件更新流程

OTA 固件更新使设备能够获取最新功能和安全修复，涉及版本检查、固件下载、分区更新等环节。

**流程描述**  
用户进入设置，选择检查固件更新。系统连接 OTA 服务器，获取最新版本信息。系统比较当前版本和最新版本。系统显示更新说明和变更内容，询问用户是否更新。用户确认后，系统下载固件到 OTA 分区。下载过程中显示进度条。下载完成后，系统验证固件完整性。系统重启，bootloader 从新分区启动新固件。新固件启动后执行必要的配置迁移。系统显示更新成功提示。

**异常处理**  
网络连接失败时提示重试。固件签名验证失败时拒绝安装并报错。下载过程中断时保留已下载部分，重试时续传。更新失败时 bootloader 自动回滚到前一版本。更新后配置格式不兼容时执行自动迁移。

## 6 数据交互接口

### 6.1 内部模块接口

**HAL 层接口**  
HAL 层向上层提供统一的硬件抽象接口，包括显示接口（HalDisplay）、GPIO 接口（HalGPIO）、电源接口（HalPowerManager）、存储接口（HalStorage）、系统接口（HalSystem）。各接口均采用 C++ 类的形式定义，包含初始化、反初始化、状态查询、操作执行等方法。

**电子书解析接口**  
电子书解析模块向上层阅读器提供统一的文件解析接口。核心类包括：Epub 类提供 EPUB 文件的加载、元数据获取、内容读取等方法；Txt 类提供纯文本文件的加载和分页方法；Xtc 类提供 XTC 格式文件的加载和分页方法。所有解析器均支持缓存机制，可选择是否使用缓存加速加载。

**渲染引擎接口**  
渲染引擎（lib/GfxRenderer）提供页面渲染接口，接收页面数据（文本、图片），输出帧缓冲数据。核心方法包括：init() 初始化渲染引擎、renderPage() 渲染指定页面、refresh() 执行屏幕刷新、clear() 清空帧缓冲。渲染引擎内部管理字体缓存和图像解码器。

**网络服务接口**  
网络服务以独立模块形式提供：CrossPointWebServer 类提供 HTTP 服务器管理；WebDAVHandler 类处理 WebDAV 请求；KOReaderSyncClient 类封装 KOReader API 调用；OpdsParser 类提供 OPDS feed 解析；HttpDownloader 类提供通用下载功能。

### 6.2 外部系统接口

**HTTP REST API**  
HTTP 服务器提供以下 RESTful 接口供外部客户端调用：

设备状态接口 GET /api/status 返回 JSON 格式的设备状态信息。文件列表接口 GET /api/files 接受 path 参数，返回指定目录的文件列表。文件上传接口 POST /upload 接受 multipart/form-data 格式的请求体，包含文件和目标路径参数。目录创建接口 POST /mkdir 接受 name 和 path 参数。文件删除接口 POST /delete 接受 path 参数。

所有接口的响应均为纯文本或 JSON 格式，错误响应包含错误代码和描述。

**WebDAV 协议**  
WebDAV 处理器实现 RFC 4918 定义的协议子集，支持标准 WebDAV 客户端的访问。协议交互基于 HTTP 1.1，认证方式为 Basic Auth（可选启用）。资源路径映射到 SD 卡文件系统路径，URL 编码的路径自动解码。

**KOReader Sync API**  
KOReader 同步服务的 API 端点定义如下：

认证接口 GET /users/auth 验证账户凭证，响应包含认证状态。进度查询接口 GET /syncs/progress/:document 获取指定文档的阅读进度，响应包含位置信息、时间戳、设备信息。进度更新接口 PUT /syncs/progress 更新阅读进度，请求体包含文档哈希、位置信息、设备信息。认证采用 HTTP Basic Auth 或自定义认证头（x-auth-user、x-auth-key）。

**OPDS 协议**  
OPDS 目录采用 Atom feed 格式，条目（entry）包含书籍元数据（标题、作者、出版日期）、链接（acquisition link）指向实际文件或详细信息页。设备支持的获取方式（acquisition types）包括 BUY、PURCHASE、DOWNLOAD、BORROW、SAMPLE。服务器应支持 Gzip 压缩传输以节省带宽。

### 6.3 数据存储格式

**配置存储**  
配置数据以 JSON 格式存储在 /settings.json 文件中，包含 settings 和 state 两个顶层对象。设置项使用枚举值存储以节省空间，字符串项使用 UTF-8 编码。配置文件的读写通过 JsonSettingsIO 类封装，确保原子性写入。

**元数据缓存**  
EPUB 电子书的解析结果缓存到 /XTCache/ 目录下的二进制文件中。缓存文件名基于原始文件路径的哈希值，避免文件名冲突。缓存文件包含 book.bin（元数据、spine、toc）和多个 section.bin（分页数据）。缓存失效条件包括原始文件修改时间变更、缓存文件损坏、系统配置变更（语言、字体等）。

**认证信息存储**  
WiFi 密码、KOReader 账户、OPDS 认证等敏感信息使用混淆算法（ObfuscationUtils）加密存储。混淆算法基于固定密钥的简单 XOR 操作，适合低安全场景下的基本保护。存储文件位于 /auth/ 目录。

## 7 权限控制策略

### 7.1 操作权限模型

本项目作为嵌入式设备固件，权限控制主要面向设备物理操作和配置变更，而非多用户认证。

**物理操作权限**  
设备上的按钮操作无需认证，任何持有设备的人均可操作。电源按钮长按可强制重启设备。SD 卡拔出可直接中断正在进行的操作。

**配置变更权限**  
系统设置、WiFi 配置、账户配置等均可在设备端直接操作，无需额外认证。配置变更立即生效，部分配置需要重启设备后生效。

**网络操作权限**  
HTTP 服务器默认不启用认证，任何可访问设备的客户端均可操作文件。WebDAV 处理器支持可选的 Basic Auth 认证，管理员可选择启用。OTA 更新接口无认证保护，建议在受控网络环境下使用。

### 7.2 安全建议

**网络隔离**  
建议将设备放置在可信的局域网环境中，避免直接暴露在互联网。若需远程访问，建议通过 VPN 连接到家庭网络。

**密码保护**  
如启用 WebDAV 认证，应使用强密码，避免使用默认密码或简单密码。WiFi 连接建议使用 WPA2 及以上加密协议。

**固件来源**  
仅从可信来源获取固件更新，避免安装来历不明的固件。OTA 更新时验证固件签名。

### 7.3 访问控制实现

**文件访问控制**  
Web 服务器自动过滤系统目录（.Spot-V100、.Trashes、XTCache）和隐藏文件（.开头）。WebDAV 处理器提供 isProtectedPath() 方法防止访问系统文件。

**接口访问控制**  
各 HTTP 接口在处理请求前检查参数有效性，防止路径遍历攻击。文件上传接口限制文件大小和类型，防止恶意文件注入。

## 8 性能指标

### 8.1 响应时间指标

**页面加载时间**  
首次打开 EPUB 书籍的加载时间应不超过 5 秒（针对 10MB 以内的常见书籍）。缓存命中后的加载时间应不超过 500 毫秒。TXT 文件的加载时间应不超过 1 秒。XTC 文件的加载时间应不超过 200 毫秒。

**翻页响应时间**  
翻页操作从按钮按下到新页面显示完成的时间应不超过 200 毫秒。局部刷新模式下应不超过 150 毫秒。

**网络操作时间**  
WiFi 连接建立时间（从选择网络到连接成功）应不超过 10 秒。HTTP 文件列表接口响应时间应不超过 500 毫秒。HTTP 文件上传速度应不低于 500 KB/s（受限于 SD 卡写入速度和网络条件）。KOReader 同步操作（查询或更新）响应时间应不超过 3 秒。

### 8.2 资源占用指标

**内存占用**  
固件运行时堆内存占用应不超过 200 KB。页面缓存占用应不超过 100 KB。字体缓存占用应不超过 50 KB。空闲内存应保持在 100 KB 以上以应对突发操作。

**存储占用**  
固件二进制文件大小应不超过 4 MB（代码段）。单本书籍缓存大小取决于书籍页数，平均每页占用 10-20 KB。配置和状态文件占用不超过 10 KB。

### 8.3 功耗指标

**静态功耗**  
设备深度睡眠时功耗应不超过 10 μA。WiFi 热点模式运行时功耗应不超过 150 mA。WiFi 站点模式（接收数据）运行时功耗应不超过 100 mA。

**动态功耗**  
电子墨水屏全屏刷新时峰值功耗应不超过 50 mA。页面渲染时功耗应不超过 30 mA。SD 卡读写时功耗应不超过 50 mA。

### 8.4 可靠性指标

**运行稳定性**  
设备应能连续运行 24 小时以上无内存泄漏导致的死机。文件系统操作应保证断电不损坏文件系统结构。OTA 更新失败时应能自动回滚到前一版本。

**数据完整性**  
文件上传和下载应保证数据完整性，失败时自动检测并报告。配置保存应保证原子性，避免部分写入导致的配置损坏。阅读进度保存应可靠，异常断电不应导致进度丢失。

## 9 安全合规要求

### 9.1 代码质量要求

**编译警告**  
代码应能通过 cppcheck 静态分析工具的检查，启用 --enable=all 规则。关键路径代码应消除所有警告。第三方库代码可使用 suppress 指令抑制已知警告。

**代码风格**  
代码应遵循 .clang-format 定义的格式化规则。命名规范应一致，使用有意义的标识符名称。注释应清晰说明复杂逻辑，遵循项目注释规范。

**测试覆盖**  
核心模块应有单元测试覆盖，包括电子书解析模块、渲染引擎模块、网络模块等。关键业务逻辑应有集成测试验证。

### 9.2 通信安全要求

**传输加密**  
HTTPS 请求应验证服务器证书，防止中间人攻击。敏感信息（密码、认证凭据）应通过 HTTPS 传输。WebSocket 连接建议使用 WSS（WebSocket Secure）。

**凭据保护**  
存储的密码应经过混淆或加密处理。本地存储的认证凭据不应明文暴露。凭据传输过程中应使用加密通道。

### 9.3 隐私合规要求

**数据最小化**  
系统不应收集不必要的用户数据。KOReader 同步数据存储在用户自行管理的云端服务中，设备不存储第三方服务器的用户凭据。

**日志脱敏**  
系统日志不应记录敏感信息（密码、完整文件路径等）。调试日志应在发布版本中禁用或最小化。

## 10 验收标准与测试用例

### 10.1 功能验收标准

**书籍阅读功能**

**系统设置功能**

### 10.4 兼容性验收标准

WiFi 应支持 2.4GHz 频段 802.11 b/g/n。HTTP 客户端应兼容主流 Web 服务器实现。WebDAV 应兼容 Calibre、OwnCloud 等标准实现。
